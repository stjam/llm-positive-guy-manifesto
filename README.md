# The LLMPositive Guy Manifesto

11 observations on the wrong kind of mindset shift in people who've gotten a little too into LLMs. Collected from real projects in Big Tech.

This manifesto is not anti-LLM. It is pro-awareness and conscious use. It fights magical thinking, mindless copy-paste, and people who confuse generating text with doing the work.

## Contributing

If you've run into cases of sloppy LLM use that aren't covered here, send them in. Two ways:

- Open an issue with the tag `case` and describe the pattern you've seen.
- Open a pull request adding the case to the manifesto directly.

I'll happily add good cases to the next revision.

## Why this exists

More and more documents, strategies, architectures, and emails look like real work but are really just poorly reprocessed autocomplete. The sweet poison of instant results burns not just tokens but the time of people who later get buried under the neuroslop. LLMs are quietly becoming a substitute for thinking, and the person using them doesn't even notice.

This is an attempt to name the patterns, so they get easier to spot — in others, and in ourselves.
